# Kaizen Action: Improve Seeding Performance in MasterDataSeeder

## Problem
In `src/Infrastructure/Services/MasterDataSeeder.cs`, specifically in the `SeedSpanishCitiesAndPostalCodesAsync` method, there are multiple `await _context.SaveChangesAsync();` calls inside loops that iterate over provinces and cities. Calling `SaveChangesAsync()` repeatedly inside loops causes an N+1 performance issue and significantly slows down the initial data seeding process.

## Objective
Improve seeding performance by removing the intermediate `SaveChangesAsync()` calls from inside the loops in `MasterDataSeeder.cs`, and relying on the final `SaveChangesAsync()` at the end of the method to persist all entities in a single batch.

## Scope
- Modify `src/Infrastructure/Services/MasterDataSeeder.cs`.
- Remove `SaveChangesAsync()` around lines 259, 266, 293, and 300.
- Verify tests still pass and the final `SaveChangesAsync()` on line 336 persists the changes correctly.
