# Clarification

No further clarification is required. The `SaveChangesAsync` calls in question exist solely to save context changes that are being tracked within the loop. The `Id` is not auto-generated by the database during `SaveChanges` but set using `_guidGenerator.NewSequentialGuid()`, so entities don't need a trip to the database before the batch can be completed.
